Why Algorithmic
Fairness Matters

Biased models cause real harm. FairLens makes fairness auditing accessible for any dataset, any domain, and any protected attribute.

⚖️

Disparate Impact

The 80% Rule: if a protected group's selection rate is below 80% of the majority, the model may violate EEOC regulations. FairLens auto-computes and flags this for any uploaded dataset.

📊

Statistical Parity

Measures the raw approval rate gap between groups. Near zero is ideal. FairLens visualizes this gap dynamically from your actual data — no static examples.

🎯

Equal Opportunity

Ensures qualified individuals in all groups have equal true positive rates — catching subtle bias that aggregate accuracy metrics completely miss.

🔧

Conditional Mitigation

Bias fix is applied ONLY when bias is detected. Pre-processing (Reweighing) → In-processing (Fairlearn EG) → Post-processing (per-group threshold calibration).

Fairness Science
Made Simple

Understand the metrics, math, and regulation behind AI bias.

Statistical

Disparate Impact (4/5 Rule)

DI = minority_rate / majority_rate. Must be >0.80 under EEOC law. FairLens auto-computes this from YOUR data and flags violations in real time.

Learn more →
Impossibility Theorem

Fairness Trade-offs

Chouldechova (2017): you cannot simultaneously achieve calibration + FPR parity + FNR parity when base rates differ. FairLens shows which constraint you're violating.

Read →
Mitigation

3-Stage Remediation

Pre: reweigh samples. In: ExponentiatedGradient with DemographicParity constraint. Post: per-group threshold calibration. Applied conditionally only when bias detected.
